ZyroMart v1.4.6

Highlights
- Settings persistence now syncs to Supabase-backed `user_account_state.app_preferences` (with local cache fallback) so account/security/preferences toggles survive reinstall and multi-device usage.
- Added schema support for `app_preferences` plus ensured missing `user_account_state` and `operator_preferences` tables in the live project.
- Customer checkout is now COD-only for this production phase, with full post-delivery settlement logic kept intact.
- Admin bootstrap stability improved: live `platform_admins` plus telemetry tables were ensured and admin credentials were re-provisioned in Supabase Auth.
- Continued auth/store hardening from v1.4.5 remains included (init guards, store-owner signup store metadata, live store map linkage).

Verification

- flutter test
- flutter build apk --release --flavor storefront -t lib/main.dart
- flutter build apk --release --flavor admin -t lib/admin_main.dart
